แสดงภาพรวมธุรกิจแบบ Real-time

  • ยอดรวมใบกำกับ, ชำระแล้ว, ค้างชำระ — ของวันนี้ / เดือนนี้ / ทั้งหมด
  • จำนวนลูกค้าที่ค้างชำระ พร้อม link ไปยังรายการ
  • กราฟยอดขายรายเดือน
  • ใบกำกับที่สร้างล่าสุด

สร้างใบกำกับ

  1. คลิก สร้างใบใหม่ ในเมนู
  2. เลือกลูกค้า (พิมพ์ค้นหาได้)
  3. เลือกสินค้า / ใส่จำนวน — ราคาจะดึงมาอัตโนมัติ
  4. เลือกประเภทการจัดส่ง
  5. กด บันทึก

พิมพ์ / ดู PDF

  1. กดปุ่ม พิมพ์ ที่แถวใบกำกับ
  2. กดปุ่ม เพื่อดูรูป PDF ใบกำกับ — ดูได้จากทุกเครื่อง (ระบบเก็บภาพไว้ให้อัตโนมัติเมื่อบันทึก/พิมพ์)
  3. ใบกำกับหลายหน้าจะแสดง badge "X หน้า"

แนบรายการคืนสินค้า

เมื่อเลือกลูกค้าที่มีรายการคืนค้างอยู่ ระบบจะแสดง banner เหลืองให้ติ๊กเลือกรายการคืนที่ต้องการหักออกจากใบนี้

บันทึกการชำระ

  1. เลือกใบกำกับจากรายการ หรือกด ในหน้าใบกำกับ
  2. ใส่จำนวนเงิน, วิธีชำระ, วันที่
  3. กด บันทึก

รายการค้างชำระของลูกค้า

กดปุ่ม ข้างแถวใบกำกับใดก็ได้ เพื่อดูรายการค้างชำระทั้งหมดของลูกค้านั้น พร้อมบันทึกภาพหรือคัดลอกไปแจ้งลูกค้า

ใบกำกับที่ลูกค้าเซ็นรับ

ในหน้าชำระเงินของแต่ละใบ มีปุ่ม อัปโหลด/เปลี่ยน สำหรับแนบรูปใบกำกับที่ลูกค้าเซ็นรับแล้ว — แนบได้เลยโดยไม่ต้องผูกกับการบันทึกชำระ และดูได้ทุกเครื่อง

ข้อมูลลูกค้า

  • ชื่อ, ที่อยู่, เบอร์โทร, เลขภาษี
  • ลิงก์ Google Maps + ระยะทาง
  • ยี่ห้อสินค้า (เพิ่มได้หลายยี่ห้อ)
  • บันทึกข้อความส่วนตัว

เงื่อนไขการชำระ

  • จำนวนบิลสูงสุด — แจ้งเตือนเมื่อค้างเกินจำนวนนี้
  • จำนวนวันสูงสุด — แจ้งเตือนเมื่อค้างเกินวันนี้
  • popup แจ้งเตือนอัตโนมัติทุกวัน

โหมดเลือกหลายราย

กด เลือก มุมขวาบน → ติ๊กลูกค้าที่ต้องการ → กด ลบที่เลือก

สินค้า

  • ชื่อสินค้า + ขนาดแพ็ค (unitSize)
  • ราคาเริ่มต้น (default price)

ราคาเฉพาะ

  • กำหนดราคาต่างกันตามลูกค้า
  • กำหนดราคาต่างกันตามประเภทการจัดส่ง
  • ดูประวัติการเปลี่ยนราคา
ลำดับความสำคัญราคา: ลูกค้า+จัดส่ง > ลูกค้าอย่างเดียว > จัดส่งอย่างเดียว > ราคา default

  1. กด เพิ่มรายการคืน
  2. พิมพ์ชื่อลูกค้า หรือ เลขใบกำกับต้นทาง — อีกช่องจะ auto-fill
  3. เพิ่มรายการสินค้าและจำนวน (เพิ่มได้หลายรายการ)
  4. บันทึก → รายการจะอยู่ใน status รอหักออก

เมื่อสร้างใบกำกับให้ลูกค้ารายนั้น ระบบจะแสดง banner ให้เลือกหักรายการคืนที่ค้างอยู่ออกจากใบได้ทันที

  • เพิ่มสีฝาขวดที่ใช้งาน
  • บันทึก รับเข้า (ซื้อ/ผลิต)
  • บันทึก จ่ายออก จะเชื่อมกับเลขใบกำกับ
  • ระบบหักออกอัตโนมัติเมื่อออกใบกำกับที่มีรายการฝาขวด
  • ดูยอดคงเหลือปัจจุบันแยกตามสี

  • กรองตามปี / เดือน / ลูกค้า / สินค้า
  • สรุปยอดขาย, ยอดชำระ, ค้างชำระ
  • Export เป็น CSV / PDF

จัดการผู้ใช้

  • เพิ่ม / ระงับผู้ใช้
  • กำหนดสิทธิ์ละเอียดต่อฟีเจอร์

ตั้งค่า

  • ชื่อบริษัท, ที่อยู่, โลโก้ (ซ่อนหัวกระดาษบนใบกำกับได้)
  • ชื่อเครื่องนี้ — ตั้งชื่ออุปกรณ์ (แสดงในการ์ด Firestore Sync)
  • โฟลเดอร์บันทึก PDF / สำรองข้อมูล — เลือกโฟลเดอร์ในเครื่อง
  • Auto Backup ตั้งเวลา
  • Export / Import ข้อมูล JSON
  • Archive log เก่า

ประวัติการใช้งาน

  • บันทึกทุกการกระทำที่เปลี่ยนข้อมูล (สร้าง/แก้ไข/ลบ, ชำระเงิน, นำเข้า, ตั้งค่า)
  • ดูได้ที่หน้า ประวัติ พร้อมผู้ทำและเวลา

ซิงค์ข้ามอุปกรณ์

  • ข้อมูลซิงค์อัตโนมัติผ่าน Firestore ทุกเครื่องที่ใช้บัญชีเดียวกัน
  • รูป PDF ใบกำกับและใบเซ็นดูได้จากทุกเครื่อง
  • สถานะการเชื่อมต่อแสดงที่แถบล่าง / ป้าย Firestore

Snapshots & Rollback

  • บันทึกสถานะข้อมูล ณ เวลานั้น (สูงสุด 3)
  • Rollback ข้อมูลกลับสู่ Snapshot ได้ทันที
  • Export / Import Snapshot เป็นไฟล์

วินิจฉัยระบบ

  • ตรวจสอบความถูกต้องของข้อมูล
  • วิเคราะห์พื้นที่จัดเก็บ
  • แก้ไขปัญหา Orphan data อัตโนมัติ
  • ดู JS Error log

รหัสผ่านถูก hash เก็บใน localStorage — ผู้ดูแลระบบ (Admin) สามารถ รีเซ็ตรหัสผ่าน ในหน้า จัดการผู้ใช้ ได้
ถ้าไม่มี Admin อื่นในระบบ ให้ล้าง localStorage ผ่าน DevTools แล้ว reload (ข้อมูลจะหายทั้งหมด — สำรองก่อน)

ข้อมูลเก็บใน localStorage ของเบราว์เซอร์นั้น — ถ้าล้าง browser data หรือใช้ Incognito ข้อมูลจะหาย
วิธีแก้: Export สำรองข้อมูลเป็น JSON ผ่านหน้า ตั้งค่า เป็นประจำ หรือเปิด Auto Backup รายวัน
Admin ยังสามารถสร้าง Snapshot เพื่อย้อนกลับได้อีกด้วย

localStorage มีขีดจำกัด ~5 MB — เมื่อเกิน 80% จะมี banner เหลือง, เกิน 95% จะเป็น banner แดง
วิธีแก้:
  • Export ข้อมูล JSON ไว้เป็น backup
  • Archive log กิจกรรมเก่าในหน้า ตั้งค่า
  • ลบ Snapshots เก่าในหน้า Snapshots
  • ตรวจสอบข้อมูลซ้ำด้วย วินิจฉัยระบบ

ถ้าพิมพ์แล้วฟอนต์ผิด: ตรวจสอบ Network, reload หน้าจอ แล้วลองพิมพ์ใหม่

เลขที่ใบกำกับสร้างจาก DDMMYY-XXX โดยใช้ตัวนับที่เพิ่มทีละ 1 ต่อวัน
การซ้ำอาจเกิดจากการ Import ข้อมูลซ้ำ — ตรวจสอบใน วินิจฉัยระบบ

แต่ต้องการอินเทอร์เน็ตเพื่อ: โหลด Bootstrap / Bootstrap Icons / ฟอนต์ Sarabun ครั้งแรก
ถ้าโหลด CSS/JS ไม่ได้ ระบบจะแสดง banner แดงแจ้งเตือน

รูป PDF ใบกำกับจะถูกเก็บอัตโนมัติเมื่อบันทึกหรือพิมพ์ใบกำกับ แล้วดูได้จากทุกเครื่องผ่านปุ่ม
ถ้าไม่ขึ้น:
  • ใบกำกับเก่าที่สร้างก่อนอัปเดต อาจยังไม่มีรูป — เปิดใบนั้นแล้วกดพิมพ์/บันทึกอีกครั้งเพื่อสร้างรูป
  • ตรวจสอบว่าเชื่อมต่ออินเทอร์เน็ต (ป้าย Firestore เป็นสีเขียว)

ข้อมูลซิงค์อัตโนมัติผ่าน Firestore เมื่อทุกเครื่องใช้บัญชีทีมเดียวกันและออนไลน์
ถ้าไม่ครบ:
  • ตรวจสอบป้าย Firestore บนแถบเมนูเป็นสีเขียว (เชื่อมต่อแล้ว)
  • ลองรีเฟรช/เปิดแอปใหม่บนเครื่องที่ไม่เห็นข้อมูล
  • ใบกำกับเก่ากว่า 6 เดือนจะไม่โหลดอัตโนมัติ — กด "โหลดใบกำกับเก่า" ในหน้ารายการ
  • Admin ตรวจสอบได้ที่ ตั้งค่า → Firestore Sync → ตรวจสอบการซิงค์

ระบบดึง เลขที่ใบกำกับ + ชื่อลูกค้าจากชื่อไฟล์ (เช่น 030669-001_ชื่อลูกค้า.pdf) และดึงยอด/วันที่/วิธีจัดส่งจากข้อความใน PDF
PDF ที่เป็นไฟล์สแกนหรือรูปถ่าย จะอ่านข้อความไม่ได้ — ระบบจะแสดง badge "PDF สแกน" ให้กรอกยอดเงิน/วิธีจัดส่งเอง (ชื่อลูกค้า/เลขที่ยังดึงจากชื่อไฟล์ได้)
ตรวจชื่อลูกค้าให้ตรงกับที่มีในระบบก่อนบันทึก

แถบเหลืองบนหน้าหลักเตือนเมื่อยังไม่ได้ตั้งโฟลเดอร์บันทึก PDF หรือ Local Folder Sync
ตั้งได้ที่หน้า ตั้งค่า — เลือกโฟลเดอร์ในเครื่องเพื่อสำรองไฟล์ไว้ในเครื่อง (เสริมจากการซิงค์ Firestore)
ถ้าตั้งแล้วแถบยังขึ้น ให้กดปิด (×) หรือเปิดแอปใหม่
